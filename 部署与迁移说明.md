# 孔中瞬变电磁训练系统部署与迁移说明

本文档用于把当前系统迁移到新服务器，并规范后续更新部署流程。

## 1. 推荐目录

优先保持和当前电脑一致：

```text
H:\codexdata\xunlian
H:\codexdata\envs\tem
H:\codexdata\pip-cache
H:\codexdata\pytorch-whl
H:\codexdata\training_jobs
```

如果新服务器没有 H 盘，可以改成 D 盘或其他数据盘，例如：

```text
D:\codexdata\xunlian
D:\codexdata\envs\tem
```

此时启动脚本里的 `ProjectDir`、`PythonExe`、`CodexDataDir` 参数也要同步改。

## 2. 迁移代码

把当前项目目录整体复制到新服务器：

```text
H:\codexdata\xunlian
```

必须包含：

```text
app.py
config.py
requirements.txt
core\
frontend\
deploy\
```

不要只复制 `frontend\dist`，后续更新前端还需要 `frontend\src`、`package.json`、`package-lock.json`。

## 3. 创建 Python 环境

在新服务器 PowerShell 执行：

```powershell
conda create -p H:\codexdata\envs\tem python=3.10 -y
```

后续不强依赖 `conda activate`，直接使用环境中的 Python：

```powershell
H:\codexdata\envs\tem\python.exe -c "import sys; print(sys.executable)"
```

安装后端依赖：

```powershell
H:\codexdata\envs\tem\python.exe -m pip install -r H:\codexdata\xunlian\requirements.txt --cache-dir H:\codexdata\pip-cache
```

## 4. 安装 GPU 版 PyTorch

先确认显卡和驱动：

```powershell
nvidia-smi
```

如果服务器能联网，可先使用 cu118：

```powershell
H:\codexdata\envs\tem\python.exe -m pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118 --cache-dir H:\codexdata\pip-cache
```

如果网络慢，可以在浏览器手动下载 `.whl` 文件，放到：

```text
H:\codexdata\pytorch-whl
```

然后安装：

```powershell
$wheels = Get-ChildItem H:\codexdata\pytorch-whl\*.whl | ForEach-Object { $_.FullName }
H:\codexdata\envs\tem\python.exe -m pip install $wheels
```

验证 GPU 是否可用：

```powershell
H:\codexdata\envs\tem\python.exe -c "import torch; print(torch.__version__); print(torch.version.cuda); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'NO CUDA')"
```

只有输出 `True`，训练才会真正使用 GPU。

## 5. 安装前端依赖

新服务器需要安装 Node.js。

进入前端目录：

```powershell
cd H:\codexdata\xunlian\frontend
npm install
npm run build
```

## 6. 环境检查

项目已提供检查脚本：

```powershell
powershell -ExecutionPolicy Bypass -File H:\codexdata\xunlian\deploy\check_environment.ps1
```

如果新服务器不是 H 盘：

```powershell
powershell -ExecutionPolicy Bypass -File D:\codexdata\xunlian\deploy\check_environment.ps1 `
  -ProjectDir D:\codexdata\xunlian `
  -PythonExe D:\codexdata\envs\tem\python.exe
```

检查项包括：

```text
Python 路径
FastAPI/Uvicorn/Numpy/Torch 是否可导入
CUDA 是否可用
NVIDIA 驱动
Node.js 和 npm
```

## 7. 启动后端

H 盘默认部署：

```powershell
powershell -ExecutionPolicy Bypass -File H:\codexdata\xunlian\deploy\start_backend.ps1
```

访问：

```text
http://127.0.0.1:8000/docs
```

局域网访问：

```text
http://新服务器IP:8000/docs
```

如果部署在 D 盘：

```powershell
powershell -ExecutionPolicy Bypass -File D:\codexdata\xunlian\deploy\start_backend.ps1 `
  -ProjectDir D:\codexdata\xunlian `
  -PythonExe D:\codexdata\envs\tem\python.exe `
  -CodexDataDir D:\codexdata
```

模型训练输出默认保存在：

```text
H:\codexdata\tem_model_任务ID.pt
H:\codexdata\tem_model_任务ID_scaler.json
H:\codexdata\tem_model_任务ID_history.json
```

如果使用 D 盘启动参数，则输出到：

```text
D:\codexdata\tem_model_任务ID.pt
```

## 9. 后续更新部署流程

每次代码更新后，在新服务器执行：

```powershell
powershell -ExecutionPolicy Bypass -File H:\codexdata\xunlian\deploy\update_dependencies.ps1
```

这个脚本会执行：

```text
1. 安装/更新 Python 依赖
2. 编译检查 app.py 和 core
3. 安装/更新前端依赖
4. 构建前端 dist
```

如果使用 Git 管理代码，推荐流程：

```powershell
cd H:\codexdata\xunlian
git pull
powershell -ExecutionPolicy Bypass -File H:\codexdata\xunlian\deploy\update_dependencies.ps1
```

然后重启后端和前端。

## 10. 路径配置

当前 `config.py` 支持环境变量覆盖路径。

常用环境变量：

```text
TEM_CODEXDATA_DIR       模型输出根目录，默认是项目上一级目录
TEM_OUTPUT_DIR          项目内部输出目录，默认是 xunlian\output
TEM_DATA_DIR            数据目录，默认是 xunlian\data
TEM_TRAINING_JOBS_DIR   训练任务状态目录，默认是 codexdata\training_jobs
TEM_DEVICE              默认设备，可设为 auto/cuda/cpu
```

一般不需要手动设置，`deploy\start_backend.ps1` 会自动设置。

## 11. 常见问题

### conda activate tem 找不到环境

如果环境是用路径创建的：

```powershell
conda create -p H:\codexdata\envs\tem python=3.10 -y
```

则不要使用：

```powershell
conda activate tem
```

直接用：

```powershell
H:\codexdata\envs\tem\python.exe
```

### pip 安装 torch 下载太慢

手动下载 `.whl` 到：

```text
H:\codexdata\pytorch-whl
```

再执行：

```powershell
$wheels = Get-ChildItem H:\codexdata\pytorch-whl\*.whl | ForEach-Object { $_.FullName }
H:\codexdata\envs\tem\python.exe -m pip install $wheels
```

### torch.cuda.is_available() 是 False

依次检查：

```text
1. nvidia-smi 是否正常
2. 是否安装了 CUDA 版 torch，而不是 CPU 版 torch
3. torch.version.cuda 是否有值
4. 显卡驱动版本是否支持所安装的 CUDA wheel
```

### 前端能打开但接口失败

确认后端是否启动：

```text
http://127.0.0.1:8000/docs
```

确认前端请求地址是否指向后端。开发模式下通常是：

```text
后端 8000
前端 5173
```

### 训练能跑但很慢

先用快速验证参数跑通：

```text
sample_size: 30
epochs: 3
time_channels: 20
use_prior: false
batch_size: 16
forward_batch_size: 5
device: auto
use_amp: true
```

正式训练再逐步提高：

```text
sample_size: 300+
epochs: 20+
time_channels: 46
use_prior: true
```
